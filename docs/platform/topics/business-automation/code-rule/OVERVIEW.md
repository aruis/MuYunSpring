# 编码规则

编码规则用于为业务记录生成稳定编码，并维护编码占用、序列状态和可回收编码。当前重点是动态记录生命周期自动赋码和平台门面显式取号。

## 能力范围

- 规则以 `moduleAlias/entityAlias/fieldName` 定位目标字段。
- 支持规则树保存、查看、预览和正式生成。
- 支持序列状态、编码台账和回收池。
- 动态记录新增、更新、删除生命周期可接入编码占用、释放和复用。
- 静态业务可以通过平台门面显式调用动态配置规则取号，但静态自动赋码声明和业务 hook 尚未作为最终契约开放。

## 运行边界

编码台账是当前占用事实表，不是流水日志。同一租户、同一规则、同一编码值只有一条当前事实，状态在 `ACTIVE`、`AVAILABLE`、`DISCARDED` 间流转。

可回收编码字段不能依赖普通动态字段唯一索引表达唯一性，否则软删后的编码复用会被数据库字段索引挡住。当前唯一占用由编码台账和动态生命周期校验承担。

序列分配和回收池消费已在 PostgreSQL 路径验证并发边界；非 PostgreSQL 数据库不宣称具备同等原子语义。

## 业务使用口径

- 动态业务配置编码规则后，由动态记录保存链路自动完成赋码和占用。
- 保存前如需前端预览，可通过动态记录编码预览入口获得候选编码。
- 运维人员可通过规则运维入口查看规则快照、定位序列桶、调整回收池状态、检查或释放台账。

接口摘要见 [WEB_API.md](WEB_API.md)。
