# 业务自动化

业务自动化专题承接平台业务编排能力，当前只记录稳定的动态表单闭环，不把动态场景包装成静态业务最终接入契约。

本专题包含三条能力线：

1. [编码规则](code-rule/OVERVIEW.md)：为业务记录生成、占用、释放和复用编码。
2. [导入导出](data-exchange/OVERVIEW.md)：为动态模块提供模板、导入、导出和错误文件闭环。
3. [记录联动](record-linkage/OVERVIEW.md)：把生单和回写合并描述为记录之间的业务联动。

## 当前边界

- 编码规则以 `moduleAlias/entityAlias/fieldName` 定位目标规则，支持动态记录生命周期自动赋码，也支持静态业务通过门面显式取号。
- 导入导出面向开启 `EXCHANGE` 能力的动态主实体，模板、导入和导出使用独立 Web 入口与动作语义。
- 生单首期稳定边界是“源记录生成目标记录草稿”，目标记录保存成功后登记来源影响关系。
- 回写首期稳定边界是“业务事件 -> 条件状态机 -> 目标定位 -> 字段影响 -> 受控系统态保存 -> 诊断台账”。

## 非目标

- 不在本文展开 Java 底层设计、服务拆分或推进过程。
- 不承诺静态自动赋码声明、静态导入导出、生成/回写配置管理 UI。
- 不把生单和回写混成同一种机制：生单返回目标草稿，回写由记录保存事件同步触发。

接口摘要见 [WEB_API.md](WEB_API.md)。
