# 记录联动

记录联动把生单和回写放在同一专题下描述。两者都处理动态记录之间的业务影响关系，但运行机制不同：生单生成目标草稿，回写由记录保存事件同步触发。

## 生单

生单的稳定边界是“源记录生成目标记录草稿”。

- 规则归属于源动态模块，启用后贡献记录级 `GENERATE` 托管动作。
- 规则配置通过 `/platform.module/{moduleAlias}/generation-rules` 维护，路径模块就是来源模块。
- 托管动作进入动态模块动作目录，复用动态 action 权限、数据权限、事务和审计链。
- 执行时重新读取持久化源记录，不信任请求体里的业务字段。
- 生成条件未命中时返回空草稿，不额外校验目标模块新增权限。
- 条件命中后校验目标模块 `CREATE` 权限，返回目标草稿、`batchId` 和每个草稿的 `originContext`。
- 首期不直接保存目标记录，不登记生成执行历史，不限制重复生成次数，不触发审批自动流转。

目标记录后续通过原生新增或生成草稿确认入口保存。保存成功后，平台根据请求根的 `originContext` 在同一事务内登记来源影响关系。

引用字段生单通过标准动态 Web 入口按字段触发，调用方不需要知道生成规则 actionCode。生成草稿确认入口接受一级子表 `children` envelope。

## 来源影响关系

来源影响关系是生单之后继续支撑任务判定、生成关系查询和后续回写定位的公共事实表，不是生成规则的临时会话。

稳定查询能力包括按 source、target、batch、generationRule 查询，按 source + target + rule 判断是否已生成，以及按已生成 target 反查 source。

`originContext` 是业务来源声明，不是安全 token，也不能写入动态业务字段、扩展 map 或子表行。

## 回写

回写的稳定闭环是“业务事件 -> 条件状态机 -> 目标定位 -> 字段影响 -> 受控系统态保存 -> 诊断台账和字段影响记录”。

- 动态新增事件提供 `after`，更新事件提供 `before/after`，删除事件提供 `before`。
- 回写保存通过 `WRITE_BACK` 来源进入动态保存链路，并继续触发目标模块保存事件。
- 默认 `SINGLE_HOP` 不消费由回写引发的下游回写事件，只有显式 `CASCADE` 才允许链式回写。
- 首期支持动态主记录到主记录、等值目标定位、生成关系目标定位和一层目标子表行定位。
- 状态触发支持 `ALWAYS`、`ON_ENTER`、`ON_EXIT`、`ON_CHANGE_WHILE_EFFECTIVE`。
- 字段写入支持 `COVER`、`ADD`、`SUBTRACT`。

回写不生成用户动作，也不使用生单的目标草稿机制。

回写规则配置通过 `/platform.module/{moduleAlias}/write-back-rules` 维护，路径模块就是触发模块。启用规则后由动态记录保存事件同步触发，不额外暴露用户动作入口。

## 后续增强

以下能力不属于当前收口范围：数量拆分、分组聚合、多根草稿、源子表自动加载、子行级来源事实、跨层级子表映射、源子表触发、多目标策略、审批事件触发、数据库级强幂等唯一约束、草稿 token、防伪签名和跨系统最终一致补偿。
