# 页面专题

页面专题覆盖动态低代码页面的交付和交互编排。它的目标不是新增一套页面内核，而是让已发布动态模块配置通过菜单入口、页面 bootstrap、动态 Web 和动作上下文形成可运行的前端契约。

本专题只描述页面业务线索和 Web 交付面，不展开 Java 底层设计。动态页面依赖的数据读写、权限、数据范围、审计、租户、事务、附件关系和动作执行，仍回到平台统一能力链路。

## 专题分层

| 目录 | 关注点 |
| --- | --- |
| `delivery/` | 页面交付闭环：菜单入口、bootstrap、列表查询、汇总、表单保存、附件关系、查重、引用候选、页面偏好 |
| `interaction/` | 页面交互编排：关联视图、局部编辑、动态弹窗、action block、模块任务、任务检查、evaluate 和公式即时试算 |

## 核心链路

```text
菜单节点
  -> /platform.menu/{menuId}/entry
  -> 页面 bootstrap
  -> 已发布 UI 配置 / 查询模板 / 动作区块
  -> /{moduleAlias} 动态 Web
  -> DynamicRecordService / 动态动作上下文
```

页面入口只定位模块、页面模式、客户端类型、默认 UI 配置、默认查询模板和入口参数。真实数据访问、动作授权、记录数据范围、字段保护、乐观锁和运行事件由动态运行态继续负责。

## 稳定边界

1. 在线页面只消费已发布且启用的 UI 配置和查询模板；未发布配置属于配置工作区。
2. UI 配置是页面交付真相源；旧 view descriptor 保持结构兼容，不反写页面配置。
3. 页面保存使用 `uiConfigId + record` wrapper；`uiConfigId` 只作为保存校验的 mutation metadata，不改变动态记录模型身份。
4. `record.attachments` 和附件维护接口只管理业务记录到 `fileId` 的关系；文件事实、二进制、预览和下载策略归属文件服务。
5. 交互区块只识别已收口类型，不把 `layoutJson.blocks` 扩展成任意前端组件注册表。
6. 所有业务动作必须回到动态 action 主链路，不能用前端私有 handler 或模块专用 controller 绕过权限、数据范围和审计。
7. 页面偏好只影响当前用户体验，可承载列表列宽、排序、筛选和个人化导出配置，不改变平台 UI 配置真相源。

## 关联专题

| 专题 | 关系 |
| --- | --- |
| 平台配置 | 提供模块、元数据、菜单、UI 配置、查询模板和发布快照。 |
| 动态运行态 | 提供 `/{moduleAlias}` 下的描述、查询、保存、引用、动作和 OpenAPI。 |
| 身份与权限 | 提供菜单剪枝、动作授权、记录数据范围和当前用户上下文。 |
| 附件与文件 | 页面只维护业务附件关系；文件服务负责文件事实和访问能力。 |
| 工作流与任务 | 模块任务不是审批待办；后续审批状态和工作流上下文可作为平台事实接入任务判定。 |
