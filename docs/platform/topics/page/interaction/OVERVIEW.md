# 页面交互概览

页面交互在页面交付闭环之上，补齐详情页关联视图、局部信息编辑、动态弹窗、动作区块和模块任务完成项。

## 能力定位

页面交互的稳定边界是业务交互可配置、可授权、可检查，而不是新增前端编排系统。所有交互仍应回到动态模块动作、权限、数据范围、动态记录运行态和审计链路。

首批能力：

1. 关联视图：详情记录到目标记录集合的查询入口，复用页面查询请求协议。
2. 局部信息编辑：通过内置动态 action executor 承载，限制在已发布 `localEdit` UI 配置字段范围内。
3. 动态弹窗：作为动作参数补录入口，初始化返回弹窗契约，提交仍走动态 action 主链路。
4. action block：通过详情 UI 配置声明动作按钮位置，bootstrap 按可见动作裁剪。
5. 模块任务：详情页业务完成项，由后端 check/evaluate 判定，不引入审批待办状态。
6. 公式即时试算：服务 onchange 和表单预览，基于页面当前记录 payload 返回试算结果和诊断。

## 配置入口

交互区块通过已发布详情 UI 配置的 `layoutJson.blocks` 声明。当前识别类型：

| block type | 作用 |
| --- | --- |
| `associationView` | 声明详情页关联列表区块。 |
| `localEdit` | 声明局部信息编辑动作按钮。 |
| `dialog` | 声明动态弹窗动作按钮。 |
| `action` | 声明普通业务动作按钮。 |
| `taskPanel` | 声明模块任务完成项区块。 |

bootstrap 会下发 `associationBlocks`、`actionBlocks` 和 `taskBlocks`。未发布配置、其他客户端配置和不可见 action 不进入在线响应。

## 交互链路

### 关联视图

详情区块声明 `associationView` 后，前端按 `viewCode` 调用关联查询。请求体沿用列表查询协议，目标记录输出按目标模块 LIST 语义和数据范围裁剪。

### 局部编辑

局部编辑动作使用内置 executor key `muyun.localEdit`。动作本身负责授权和记录数据范围校验；执行器只允许提交已发布 `localEdit` UI 配置中的主关系字段，并要求 `record.version` 参与乐观锁。

### 动态弹窗

DIALOG action 只负责初始化弹窗和返回提交入口。`executorKey` 可用 `dialogKey#submitActionCode` 绑定后续业务动作；实际提交仍走动态 action 主链路，避免前端私有 handler 绕过平台能力。

### 模块任务

模块任务区块由 `taskPanel` 声明。后端任务检查先校验当前租户、动作授权和当前记录 `VIEW` 数据范围，再执行完成项判定。

首期判定类型：

| 类型 | 语义 |
| --- | --- |
| `ASSOCIATION_VIEW` | 按关联视图是否命中目标记录判定。 |
| `QUERY_TEMPLATE` | 按查询模板命中结果判定，可用 `externalRecordIdKey` 注入当前记录 ID。 |
| `MANUAL` | 只返回 `UNKNOWN`，作为前端引导，不保存办理状态。 |

### 公式即时试算

公式即时试算使用动态运行态的公式 preview 入口。前端提交当前页面记录 payload，后端返回试算后的记录、变更字段和结构化诊断；该链路不落库，也不替代保存时的公式和校验执行。

## 权限边界

1. 关联查询使用 `QUERY` 动作权限，目标数据仍按目标模块查询链路和数据范围输出。
2. 局部编辑、弹窗提交和普通动作执行使用动态 action 自身的授权、记录级数据范围和 action availability。
3. 模块任务检查使用 `VIEW` 动作权限，并先确认当前记录对当前用户可见。
4. bootstrap 的 action block 裁剪只代表动作对当前用户可见；记录级可用性仍以后端 action availability 和执行入口为准。
5. 公式试算只服务页面交互反馈；字段可见、可编辑范围和最终数据正确性仍由 bootstrap、动作配置和保存链路兜底。

## 非目标

1. 不做完整 BPMN 设计器。
2. 不新增前端私有 handler、脚本执行器或模块专用 controller。
3. 模块任务不是审批待办，不保存处理人、委托、加签或办理状态。
4. 复杂条件不新增平行语言；优先通过查询模板、关联视图和后续平台事实接入。
