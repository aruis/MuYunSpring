# 工作流工作台概览

工作流工作台承接办理人日常视角：我的待办、已办、知会、发起跟踪和委托责任。它是工作流运行闭环的读侧与轻量操作入口，不替代流程定义、实例管理或业务表单页面。

## 能力定位

工作台面向当前用户聚合任务卡片和统计：

1. 待办展示当前用户需要处理的审批或业务任务。
2. 已办展示当前用户已经处理过的任务。
3. 知会展示当前用户收到的通知任务，并支持已读状态。
4. 发起跟踪展示当前用户发起的流程实例。
5. 委托视角展示当前用户因委托关系承担的任务责任。
6. 统计接口按 board 返回工作台计数。

## 卡片信息

工作台卡片围绕流程实例、业务记录和任务责任组织信息。当前后端已能透出：

| 信息 | 说明 |
| --- | --- |
| 模块与记录 | 用于定位业务记录和运行实例。 |
| 定义与版本 | 用于识别流程来源。 |
| 实例状态 | 表达运行中、结束、终止等实例状态。 |
| 任务类型与状态 | 区分审批、知会、业务任务以及待办、完成、取消、失效等状态。 |
| 分派解释 | 表达直接分派、委托分派、转办后处理权等事实。 |
| 加签来源 | 可透出 `addedByAddSign`、加签来源节点、操作者和加签时间。 |
| 超期状态 | 用于展示审批节点预警或超期标记。 |
| 知会已读 | 知会列表支持未读、已读和全部筛选。 |

## 筛选与排序

工作台查询使用白名单请求字段，不开放任意数据库字段。当前可按模块、记录、定义、版本、实例状态、节点、任务类型、任务状态、分派类型、超期状态、知会已读状态、发起时间、接收时间、完成时间、最后操作时间、到期时间、加签来源和发起人等维度筛选。

排序由 `WorkflowWorkbenchSort` 表达，具体可用字段以运行时服务白名单为准。

## 典型使用

1. 用户打开工作台，前端按 board 拉取统计。
2. 进入待办、已办、知会、发起跟踪或委托页签，按当前用户查询卡片。
3. 用户从待办卡片进入业务记录或流程渲染包。
4. 审批任务通过任务动作处理；业务任务先准备处理 bundle，再检查并继续推进。
5. 知会任务可在阅读后调用已读入口。

## 边界说明

1. 工作台只做当前用户视角聚合，不绕过任务归属和动作权限。
2. 工作台卡片不是流程实例的完整详情；完整节点、route、任务和事件仍从实例渲染包或历史入口读取。
3. 工作台不负责流程定义、委托配置管理和管理端强制处理。
4. 跨模块业务标题解析当前没有稳定门面，卡片标题增强按后续业务标题 resolver 专题推进。
