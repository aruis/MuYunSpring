# 工作流与任务专题概览

工作流与任务专题把审批节点和任务节点统一到同一套 workflow 能力中，形成定义、发布、运行、任务、待办、历史、委托和插件挂点的后端闭环。

本文只梳理业务能力线索和稳定边界，不展开 Java 底层设计。Web 入口见 [WEB_API.md](WEB_API.md)，工作台能力见 [workbench/OVERVIEW.md](workbench/OVERVIEW.md)。

## 能力定位

工作流不是只有状态字段和待办入口的简化能力。它负责把业务记录、审批治理、任务办理、分支汇聚、历史归档和动作权限串到同一条运行链路。

核心定位：

1. 统一使用 HYBRID 工作流模型，不拆成审批流和任务流两套实现。
2. 流程通过 `approvalEnabled` 声明是否治理业务记录主审批态。
3. 审批治理流程在同一模块、租户、业务记录上当前有效实例独占。
4. 审批完成节点 `APPROVAL_COMPLETED` 回写业务记录审批通过，但不等于整条流程结束。
5. 任务节点是可独立配置、判定和引导动作的业务对象，不展示审批动作。
6. 分支、汇聚和 route 运行对象共同表达流程实际路径。
7. 运行时动作、任务动作和管理动作接入平台动作权限、数据权限、事件和审计上下文。
8. 历史归档冻结实例快照、任务、事件和画布回显信息。

## 主要对象

| 对象 | 说明 |
| --- | --- |
| 工作流定义 | 流程配置入口，绑定模块、审批治理开关、版本和当前发布态。 |
| 定义版本 | 发布后的冻结版本，保存节点、连线、语义图和布局图 JSON。 |
| 节点定义 | 表达审批、知会、任务、分支、汇聚、系统动作等流程节点。 |
| 连线定义 | 表达节点间 route、默认线、手工选择和条件选择。 |
| 流程实例 | 业务记录上的当前运行对象，保存定义版本快照、实例状态和审批态。 |
| 节点实例 | 运行态节点状态，承载活动、完成、跳过、超期等事实。 |
| route 实例 | 运行态路径事实，是分支有效路径的真相源。 |
| 工作流任务 | 用户办理对象，覆盖审批、知会、业务任务、委托和转办后的处理事实。 |
| 历史实例 | 撤回、重置、终止或正常结束后的冻结归档。 |
| 委托配置 | 用户或管理端维护的任务分派规则，只在正式 TODO 任务生成时匹配。 |

## 核心业务链路

### 定义与发布

1. 维护工作流定义、节点、连线和任务定义。
2. 发布时生成定义版本，并冻结节点、连线、语义图和布局图 JSON。
3. 发布后的定义可向目标模块贡献稳定 runtime 动作目录。
4. 运行实例只读发布快照；运行态加签只更新实例级画布快照，不反向修改定义版本。

定义和版本配置按模块聚合在 `/platform.module/{moduleAlias}/workflow-definitions` 下。普通维护只面向草稿；发布、停用和归档通过发布门面同步定义状态和模块动作贡献。运行态审批、任务办理和管理动作仍归属 `/workflow/runtime` 等运行入口。

### 提交与实例

1. 业务记录可读取提交状态和提交预览。
2. 提交审批时选择已发布且匹配的工作流定义。
3. `approvalEnabled=true` 时，当前有效审批治理实例在同一业务记录上独占。
4. 实例启动后生成节点、route 和任务快照，并按流程推进。
5. 到达 `APPROVAL_COMPLETED` 后回写业务审批通过；后续节点仍可继续运行。

### 审批与任务

审批节点支持同意、驳回、重提、撤回、线性回退、分支域回退、转办、加签、强制同意和任务失效/取消等动作。知会任务不承载审批动作，提供已读入口。

任务节点用于业务完成项引导和检查。业务任务可先准备处理 bundle，再执行 `check-and-continue`；后端完成静默通过、人工确认、动作后判定和后续分支选择校验。

### 分支与汇聚

分支通过 route 表达实际有效路径。自动分支按规则和默认线推进；手工分支要求在本轮可达候选出线中显式选择，可携带选择原因。汇聚节点按已选路径判定是否继续推进。

当前已覆盖默认线、结构化多 MANUAL 分支选择、路径裁剪、有效分支域内回退和 route 原因回显。嵌套分支、汇聚后复杂回退等场景仍按后续真实业务扩展。

### 委托、转办与加签

委托只在正式 TODO 任务生成时匹配，不改变定义和实例图，不追链。任务会冻结原处理人、委托人、被委托人、策略和处理权解释。转办不会重新匹配委托，并切断原委托人额外处理权。

加签按运行态图编辑插入局部单出口收敛段，可包含加签段内部审批、分支和汇聚节点。同一来源节点已有未生效加签段时允许替换；已生效或已生成任务的加签段冻结后拒绝修改。

### 终止、重置与历史

普通终止、强制终止、撤回和重置会迁移或冻结实例事实，并释放当前审批绑定。历史实例提供列表、渲染包、任务、事件和解释视图；管理端可查询和删除历史实例。

### 超期与预警

超期能力只扫描活动审批节点，只更新节点超期状态并写入预警或超期事件。不自动审批，不自动流转。

### 插件挂点

工作流运行插件面向稳定动作前后切点。当前支持 `submit`、`approve`、`forceApprove`、`transfer`、`reject`、`rollback`、`revoke`、`reset`、`terminate`、`forceTerminate` 的 before/after 事件。

插件可同步执行或 after-commit 执行；默认同步异常会阻断主流程。改写实例图的 `addSign` 不开放普通插件，异步分发、插件配置管理和动态脚本后续再展开。

## 边界说明

1. 本专题以当前后端闭环为边界，不补完整前端流程设计器。
2. 语义图和布局图 JSON 用于回显，不参与流程判定。
3. 工作流动作复用平台动作、事件、审计上下文和身份权限，不另起一套权限体系。
4. 模块任务不是工作流审批待办；它不保存处理人、办理状态、委托或加签。
5. 跨模块业务记录标题解析当前没有稳定门面，不在本专题硬补。
6. workflow 发布贡献的 runtime 动作目录保持模块级稳定可授权，不随单个定义停用自动回收。
7. 管理端强制终止、强行代办、管理查询和历史删除归属 `platform.workflow_admin` 管理模块动作。

## 关联专题

| 专题 | 关系 |
| --- | --- |
| 动静一体核心 | 工作流状态字段、实例字段和任务字段要兼顾静态模型与动态元数据模型。 |
| 平台执行底座 | 工作流动作复用动作上下文、规则引擎、事件和审计上下文。 |
| 身份与权限 | 工作流运行、工作台和管理动作复用当前用户、动作权限和数据权限。 |
| 动态运行态 | 动态记录通过模块动作目录提交审批、执行工作流动作和读取运行详情。 |
| 页面交付 | 流程画布、工作台卡片和表单任务面板属于前端编排面。 |
| 业务自动化 | 回写、生成、导入导出等专题可通过工作流挂点进入业务编排。 |
