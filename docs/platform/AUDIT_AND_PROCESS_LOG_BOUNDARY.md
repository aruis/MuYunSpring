# 审计与专题流水边界

本文用于统一判断平台运行审计和各业务专题流水的职责边界，避免把所有过程记录都塞进同一张平台审计表。

平台运行审计承担轻量留痕和排查线索；专题流水承担业务过程事实。两者可以通过 `traceId`、`moduleAlias`、`recordId`、任务 ID、批次 ID 等字段互相定位，但不互相替代。

流程委托代办归工作流任务闭环，职员业务代办归身份权限闭环；两者只在运行审计中统一记录必要操作者线索，不自动合并业务模型或处理链路。

## 判断原则

| 问题 | 归属 |
| --- | --- |
| 谁在什么租户、什么模块、什么动作下做了操作，授权结果如何 | 平台运行审计 |
| 代办场景下实际操作者和被代办业务主体是谁 | 平台运行审计 |
| 业务记录在流程中经过哪些审批节点、任务、委托、转办、加签和历史归档 | 工作流专题流水 |
| 导入执行解析了多少行、新增/更新/跳过/失败多少、错误文件如何下载 | 导入导出专题结果或批次 |
| 配置包发布、回滚、迁移 dry-run、导入草稿和健康门禁结果 | 配置治理专题事实 |
| 生单来源关系、回写贡献、失败诊断和后续定位 | 记录联动专题事实 |
| 后台任务、批处理、外部写入或系统态操作的执行批次、重试和失败明细 | 对应专题或运维治理流水 |

## 平台运行审计

平台运行审计围绕 `RuntimeEvent` 和 `RuntimeAuditRecord` 工作，记录动作和记录事件的必要上下文：

1. 事件、链路、模块、实体、记录、动作和发生时间。
2. 当前租户、系统态标记、操作者和授权判定。
3. 动作结果的轻量摘要，例如执行器、动作级别、结果类型、错误阶段和错误类型。
4. 代办场景的委托 ID、被代办账号、职员、机构、部门和任岗 ID。

运行审计不承载表单字段快照、业务状态差异、审批意见正文、导入行数据、外部报文全文或重试过程明细。需要这些事实时，应由对应专题建立自己的模型、保留策略和查询入口。

## 专题流水

专题流水只有在确实需要表达业务过程事实时才建设。判断依据是：没有它，业务无法解释、追责、恢复或继续推进，而不仅仅是排查一次动作是谁触发的。

当前口径：

| 专题 | 是否需要专题流水 | 说明 |
| --- | --- | --- |
| 动态 CRUD 和普通动作 | 不默认建设 | 使用平台运行审计即可；业务字段变化不做通用快照。 |
| 业务代办 | 不单独建设 | 委托关系是身份权限事实；动作留痕进入运行审计。工作流审批接管另归工作流。 |
| 工作流与任务 | 需要 | 实例、节点、route、任务、事件和历史是流程运行真相源。平台审计只记录动作入口和操作者线索。 |
| 导入导出 | 需要轻量结果，暂不做重型通用流水 | 当前结果、错误文件和 token 足够支撑交互；后续出现异步批次、重试、恢复或审计留档时，再建设导入批次模型。 |
| 记录联动 | 需要 | 来源影响关系、回写贡献和失败诊断是后续定位和回放的业务事实。 |
| 配置治理 | 需要 | 发布版本、回滚、健康报告、dry-run 和导入草稿是配置迁移治理事实。 |
| 页面交互计算 | 不建设 | onchange、预览和 evaluate 等只做即时计算，不保存业务流水。 |
| 外部写入 | 待业务触发 | 如只调用标准动作，进入运行审计；如需要幂等、重试、外部回执和报文留存，按 [外部写入接入边界](EXTERNAL_WRITE_BOUNDARY.md) 建设接入专题流水。 |
| 后台任务和批处理 | 待业务触发 | 系统态动作进入运行审计；批次进度、重试、失败恢复和执行明细由后续统一后台任务调度或运维治理专题承接。 |

## 建设约束

1. 新专题流水必须有明确业务对象和保留策略，不用平台审计表当万能 JSON 仓库。
2. 专题流水优先记录摘要、状态、关联 ID、错误类型和必要诊断，不默认保存完整业务内容。
3. 能通过运行审计定位到专题流水时，不重复记录操作者、权限判定和动作上下文。
4. 能通过专题流水解释业务过程时，不要求平台审计还原业务过程。
5. 外部报文、导入文件、审批意见等可能包含敏感信息的内容，应由专题按脱敏、下载权限和保留期单独治理。

## 后续触发

以下场景出现时，再单独拆战役建设对应流水，不提前做通用事件流：

1. 导入执行改为异步批次，需要查询进度、重试和恢复。
2. 外部系统写入需要幂等键、回执、重试、报文摘要和失败补偿，按 [外部写入接入边界](EXTERNAL_WRITE_BOUNDARY.md) 拆分接入模型。
3. 后台任务需要统一调度、运行批次、重试策略和失败恢复。
4. 某类业务单据需要长期保留状态流转、字段变更或业务事件解释视图。
5. 平台运行审计需要归档、保留期、脱敏查询或跨租户安全审查。
