# M7 稳定契约

本文记录 M7 平台业务编排能力的收口契约，服务于后续 M8 业务专题接入。这里不记录推进过程，只记录已经稳定的运行边界、测试支撑和明确后续项。

## 收口范围

M7 第一批能力包括编码规则、导入导出、推送生成和回写。当前四个能力块均达到可收口边界：

1. 编码规则提供规则模型、预览、正式生成、序列状态、台账、回收池、动态记录生命周期接入和静态显式调用门面。
2. 导入导出提供动态模块模板、解析、执行、错误文件、子表导出、引用 title 往返和 Web/OpenAPI 暴露面。
3. 推送生成提供动态记录之间从规则、托管动作、权限、目标草稿、来源声明到来源影响关系登记的闭环。
4. 回写提供动态记录事件驱动的规则执行、同步阻断、目标定位、字段影响、贡献台账和失败诊断闭环。

M7 当前只承诺动态表单闭环。静态业务后续可以复用平台能力，但不把当前动态场景包装成静态最终接入契约。

## 推送生成契约

推送生成的稳定边界是“源记录生成目标记录草稿”。规则归属于源动态模块，启用后贡献记录级 `GENERATE` 托管动作。动作目录只承载托管动作和权限入口，`generationCondition` 不作为动作可用性表达式下发，避免动作可用性语义和生成运行时公式语义漂移。

执行顺序如下：

1. 动态 action 链路校验源记录动作权限和数据权限。
2. 运行态重新读取持久化源记录，不信任请求体里的业务字段。
3. `generationCondition` 按持久化源记录快照判断；不命中时返回空草稿，不额外校验目标模块新增权限。
4. 条件命中后校验目标模块 `CREATE` 权限。
5. 返回目标模块草稿、`batchId` 和每个草稿的 `originContext`。

首期不直接保存目标记录，不登记生成执行历史，不限制重复生成次数，不触发审批自动流转。目标模块后续通过原生新增或更新保存链路提交草稿；保存成功后，如果请求根携带 `originContext`，平台在同一事务内登记来源影响关系。

当前草稿能力支持：

1. 单个目标根草稿。
2. 已加载一级源子表到目标子表草稿映射。
3. 一个已加载子表 `splitDriver` 逐行生成多个目标根草稿。
4. split driver 行可映射到目标根，也可为每个目标根附带一个目标子行。

当前明确拒绝根对象 split driver、多个 split driver、数量拆分、分组聚合和多子 mapping 组合。

## 来源影响关系契约

来源影响关系是推送生成之后支撑任务判定、生成关系查询和后续回写定位的公共事实表，不是生成规则的临时会话。

稳定查询能力包括：

1. 按 source 查询。
2. 按 target 查询。
3. 按 batch 查询。
4. 按 generationRule 查询。
5. 按 source + target + rule 判断是否已生成。
6. 按已生成 target 反查 source，用于回写生成关系定位。

`originContext` 是业务来源声明，不是安全 token，也不能写入动态业务字段、扩展 map 或子表行。运行态只把它作为动态保存的 mutation metadata 传递。草稿 token、防伪签名和跨系统最终一致补偿不属于当前收口范围。

## 回写契约

回写的稳定闭环是“业务事件 -> 条件状态机 -> 目标定位 -> 字段影响 -> 受控系统态保存 -> ExecutionLog/EffectLog”。

动态记录新增、更新和删除会产生事务内 mutation event：

1. 新增保存事件提供 `after`。
2. 更新保存事件提供 `before/after`。
3. 删除事件提供 `before`。
4. 事件携带 `mutationSource`、`traceId`、`depth`、`parentExecutionId`、`cascadeAllowed` 和 metadata。

回写保存通过 `WRITE_BACK` 来源进入动态保存链路，并继续触发目标模块保存事件。默认 `SINGLE_HOP` 不消费由回写引发的下游回写事件，只有显式 `CASCADE` 才允许链式回写。

规则执行首期支持：

1. 动态主记录到主记录。
2. 等值目标定位。
3. 生成关系目标定位。
4. 一层目标子表行定位。
5. `ALWAYS`、`ON_ENTER`、`ON_EXIT`、`ON_CHANGE_WHILE_EFFECTIVE` 触发模式。
6. 多个状态触发模式组合，但 `ALWAYS` 不能与状态触发组合。
7. `COVER`、`ADD`、`SUBTRACT` 字段写入。

目标子表行定位通过规则声明 `targetRelationCode/targetEntityAlias`。命中目标子行后，运行态使用目标根记录的局部子表受控保存链路更新子行，并保留 relation child lifecycle，不用直接子行 update 绕过主子关系语义。

## 回写台账契约

`ExecutionLog` 记录规则执行诊断，支持按 trace、rule、触发记录、目标记录和失败状态查询。`planned`、`success` 和 `no-op` 状态随业务事务提交；目标保存失败时通过独立事务保留已定位目标和字段 patch 快照。

`EffectLog` 记录目标保存成功后的字段级影响事实，包括 execution、trace、触发记录、目标记录、目标字段、来源字段、操作、状态、贡献值、数值 delta 和 before/after 文本值。

数值贡献规则遵循以下边界：

1. `ADD/SUBTRACT` 必须绑定状态触发。
2. 进入生效状态时写全量贡献。
3. 退出生效状态时基于 active EffectLog 撤销历史贡献。
4. 生效期间来源值变化时基于 active EffectLog 只写新旧贡献差额。
5. 重复进入、重复退出等重复事件通过 active 贡献状态转为 no-op，不重复修改目标字段。
6. 非贡献型字段基于 APPLIED EffectLog 识别同 rule、触发记录、目标记录、目标字段、来源字段和 afterValue 的重复执行，并转为 no-op。

当前没有承诺数据库级唯一幂等约束。

## 测试边界

M7 收口由以下测试族支撑：

1. 编码规则 contract/IT 覆盖预览、正式生成、序列状态、台账、回收池、动态生命周期、静态门面和并发边界。
2. 导入导出测试覆盖模板、导入、导出、错误文件、子表导出、引用 title 往返和 OpenAPI 门控。
3. 推送生成测试覆盖托管动作贡献、权限校验、条件判断、目标草稿、一级子表草稿、split driver、来源声明和生成关系定位回写。
4. 来源影响关系测试覆盖来源登记、查询门面、重复来源关系和 origin metadata 转换。
5. 回写测试覆盖规则保存、触发模式归一化、目标定位、子表局部保存、状态贡献、重复 no-op、失败诊断、系统态查询和真实动态保存入口。
6. Web 动态记录测试覆盖 `originContext` 进入 mutation metadata、`children` envelope 解析和非法子表输入 fail-fast。

默认收口验证命令是：

```bash
./gradlew test
```

## 后续增强池

以下能力已经明确为后续项，不阻塞 M7 收口：

1. 推送生成数量拆分、分组聚合、多根草稿、源子表自动加载、子行级来源事实和跨层级子表映射。
2. 回写源子表触发、多目标策略、审批事件触发和数据库级强幂等唯一约束。
3. 生成/回写配置 Web 管理入口、链路图谱、前端审计页和业务可视化治理。
4. 草稿 token、防伪签名、`originContext` 安全治理和跨系统最终一致补偿。
5. `RecordGenerationActionExecutor` 与 `RecordWriteBackRuntimeService` 后续按 planner、locator、ledger、builder 等职责做结构拆分。

后续增强必须由明确业务场景驱动，不应为了兼容旧系统复制历史包结构、历史字段或临时前端协议。
